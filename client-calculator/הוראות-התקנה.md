# מחשבון רווחיות ללקוח — הוראות התקנה

אפליקציה שמקושרת **חי** לגיליון הגוגל שיטס שלך. הלקוח ממלא פרמטרים ומקבל את התוצאות —
**בלי לראות את הנוסחאות** (הן נשארות בגיליון, על שרתי גוגל).

---

## איך זה עובד (בקצרה)

1. הלקוח פותח קישור, ממלא את הפרמטרים (ריבית, שכירות, גודל חיבור וכו').
2. האפליקציה כותבת את הקלט לתאים `B2:B9` בגיליון, נותנת לגיליון לחשב עם הנוסחאות שלך.
3. קוראת את התוצאות ומציגה ללקוח — ואז **משחזרת את ערכי ברירת המחדל** בגיליון.
4. **סדר התצוגה:** קלט → שדות → פיננסי למשקיע → תשואה (IRR) → פיננסי לפרויקט → (שפיצר).
5. **התצוגה תלויה ב-`B8` (אחוז שותף):**
   - `B8` = 0 או ריק → **לקוח רגיל**: שדות + פיננסי למשקיע + פיננסי לפרויקט + **שפיצר פרויקטלי בלבד**
     (אקורדיון, בלי כפתור מעבר). **אין תשואה, אין שפיצר פרטי.**
   - `B8` > 0 → **שותף**: כנ"ל, **ובנוסף** כרטיס תשואה **Private Leveraged IRR (CoC)** (מתא `B31`),
     והשפיצר נפתח על **פרטי** כברירת מחדל עם כפתור מעבר לפרויקטלי. השפיצר תמיד כולל את **כל השנים** לפי `B5`.
   - התוכן הפרטי (IRR + שפיצר פרטי) כלל לא נשלח מהשרת ללקוח רגיל.

### מיפוי התאים (בקובץ `Code.gs`)
| נתון | תא/טווח |
|------|---------|
| קלט הלקוח | `B2:B9` |
| שדות (פנלים + אגירה) | `A34:C35` |
| פיננסי למשקיע | `A25:C30` |
| תשואה — Private Leveraged IRR (CoC) | `B31` (רק לשותף) |
| פיננסי לפרויקט | `A16:C22` |
| שפיצר (כל השנים) | `E2:Q52` (מסונן לפי שנים תקינות) — פרויקטלי `E–L`, פרטי `E,F,M–Q` |
| סטטוס שותף | `B8` |

> הנוסחאות אף פעם לא יורדות למחשב של הלקוח. גם מפתח שיבדוק את הדף יראה רק מספרים.

## התקנה (חד-פעמי, ~5 דקות)

### 1. פתיחת עורך הקוד
בגיליון: **Extensions → Apps Script** (בעברית: תוספים → Apps Script).

### 2. הדבקת הקוד
- במסך שנפתח, מחק את מה שיש בקובץ `Code.gs` והדבק את התוכן של **`Code.gs`** מהתיקייה הזו.
- לחץ על **+** ליד "Files" → **HTML** → תן לו שם **`Index`** (בדיוק ככה, באות גדולה).
  מחק את התוכן ההתחלתי והדבק את **`Index.html`** מהתיקייה הזו.
- שמור (אייקון הדיסקט / `Ctrl+S`).

### 3. פרסום כאפליקציה
- למעלה מימין: **Deploy → New deployment**.
- ליד "Select type" לחץ על גלגל השיניים → **Web app**.
- הגדרות:
  - **Execute as:** `Me` (אתה) — זה מה שמאפשר לאפליקציה לגשת לגיליון בשמך.
  - **Who has access:** `Anyone` — כדי שהלקוח יוכל להיכנס בלי חשבון גוגל וללא גישה לגיליון.
- **Deploy** → תתבקש **לאשר הרשאות** (Authorize) — אשר עם החשבון שלך.
  (ייתכן מסך "Google hasn't verified this app" → "Advanced" → "Go to … (unsafe)". זה בסדר, זו האפליקציה שלך.)
- העתק את ה-**Web app URL** — זה הקישור שנותנים ללקוח. ✅

### 4. עדכון בעתיד
- **שינוי נוסחה?** פשוט ערוך בגיליון — האפליקציה מתעדכנת אוטומטית, בלי לגעת בקוד.
- **שינוי בעיצוב/קוד?** ערוך ב-Apps Script → **Deploy → Manage deployments → עריכה (עיפרון) → Version: New version → Deploy**.
  (הקישור נשאר אותו קישור.)

## חשוב לדעת

- **תאי הקלט** הם `B2:B9` בלבד — אל תשים בהם נוסחאות (הם נכתבים ומשוחזרים בכל חישוב).
- אם המבנה בגיליון זז (שורות/עמודות) — כוונן את הטווחים בראש הקובץ `Code.gs` (מסומנים בהערות).
- **ה-IRR** (Private Leveraged IRR CoC) מגיע מתא `B31` ומוצג רק לשותף. אם המספר יושב בתא אחר — עדכן את `PRIVATE_IRR_CELL` ב-`Code.gs`.
- **ריבוי לקוחות:** יש נעילה (LockService) שמונעת התנגשות בין חישובים מקבילים. מתאים לעומס רגיל.

### הלוגו
הלוגו האמיתי של mesh (`logo.jpg`) מוטמע ב-`Index.html` כ-`<img class="logo" src="data:image/jpeg;base64,…">` —
עצמאי לחלוטין, ללא צורך באחסון חיצוני. כדי להחליף אותו בעתיד: הרץ מחדש את `embed_logo.py`
(או החלף ידנית את ה-`src` של `<img class="logo">`) ופרסם מחדש.
